The Mediator Pattern in Angular Templates

The Mediator Pattern defines an object (the mediator/component) that controls how a set of objects interact. Instead of objects communicating directly with each other, they communicate through the mediator.

In This Example:

  • Colleagues: UI elements (input field, buttons, message list, user list, displays)
  • Mediator: The ChatRoomComponent (click to see component code for details)
  • Communication: Input -> Component -> Output (not direct element-to-element communication)

This maintains loose coupling and makes the code easy to extend. New UI elements can be added without modifying existing ones!

How the Mediator Pattern Works Here

Data Flow Through the Mediator:

1. Typing Detection

Input Field (Colleague #1)

↓

Component.onMessageInput() (Mediator)

↓

Typing Indicator (Colleague #2) Updates

Input field doesn't know about typing indicator. Component coordinates.

2. Message Sending

Send Button (Colleague)

↓

Component.sendMessage() (Mediator)

↓

Multiple Updates: Messages List + Input Field + Typing Indicator + Count Badge

Button doesn't communicate with messages, input, or indicators. Mediator handles all!

3. User Selection

User List Item Click (Colleague #4)

↓

Component.selectUser() (Mediator)

↓

Chat Header + Message Display Update

User list doesn't know what displays will change. Mediator controls it.

Benefits of the Mediator Pattern:

  • Loose Coupling: UI elements don't depend on each other
  • Centralized Logic: All interactions managed in the component
  • Easy to Extend: You can add new UI elements without modifying existing ones
  • Single Responsibility: Each element focuses on its own responsibility
  • Testable: Easy to test component logic in isolation
