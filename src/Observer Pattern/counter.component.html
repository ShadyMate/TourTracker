Observer Pattern in Angular Signals

Components:

  • Subject (counterSignal): The signal in CounterService holds mutable state. When it changes, all observers are automatically notified.
  • Observers (computed signals, effects, template): They watch the subject and automatically react to changes.
  • Update Propagation: When increment() is clicked, the signal updates and automatically notifies all dependents. Check console for the effects.

How It Works:

  1. Component reads signals in the template: {{ counter }}
  2. Angular tracks these "reads" as dependencies
  3. When signal value changes, Angular knows which parts depend on it
  4. Only affected observers (computed, effects, template) update
  5. No zone.js-based change detection needed - fine-grained reactivity!

Check the browser console to see effects running automatically!
